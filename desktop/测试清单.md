## 11. 固件升级（2026-06-18 新增 · 串口刷写 MVP）

> 视觉真值 `..\web\mockups\desktop-firmware.html`（浏览器双击，有六态切换条）。
> 需先装新依赖：`python -m pip install -e ".[dev]"`（已加 esptool/pyserial）。

- [ ] 侧栏「固件升级」导航项（升箭头图标）；进入页见「设备当前版本 → 升级到版本」双 tile。
- [ ] 升级到版本显示 **v1.0.0**（直接从 .bin 烤进的版本读，非文件名）；「升级固件」行显示随包文件名 `clawd-mochi-app-v1.0.0.bin`，「选择文件…」可改用下载的 .bin（版本随之更新；把文件改个名版本仍正确）。
- [ ] 设备在线（同局域网 + 已刷含 `/info` 的新固件）时当前版本读自 `/info`，否则「未知」且不影响刷写。
- [ ] 串口下拉列出本机串口，疑似 ESP（Espressif/CH340/CP210x/JTAG）标 ★ 并排在前；「↻ 刷新」可重列。
- [ ] 无串口时：按钮置灰 + 黄横幅（提示用数据线 / CH340 驱动）。
- [ ] 用 USB 线连设备 → 选口 → 「刷写固件」：进度条推进、黑底日志滚动、刷写中按钮禁用且提示「请勿拔线」。
- [ ] 刷写成功：绿横幅「已重启」，设备亮屏走开机问候；几秒后当前版本应变 v1.0.0。**WiFi 配置应仍在**（写 0x10000 不碰 NVS）。
- [ ] 串口被占用 / 拔线中断：红横幅 + 原始错误，可重试。
- [ ] 自动化：`python -m pytest -q` 应 **83 passed**（含 firmware/flash_runner 37 例）。

> 单文件 = app 镜像 `clawd-mochi-app-v<版本>.bin`，写 **0x10000**（保留 WiFi/情绪）。别人下载该 .bin → 本页「选择文件…」选它升级。
> **全新空白芯片**首次需 IDF 工具链完整烧录（`_build_idf.bat -p COMx flash`）；之后单文件升级即可。
