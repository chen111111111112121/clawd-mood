# Clawd Mochi 全状态测试指令

用于验证设备 6 种 Monitor 状态是否正常响应。测试前请确认：

1. 固件已烧录（含 `/status` 端点）
2. 设备已连上家里 WiFi，屏幕或 Web 页显示局域网 IP
3. PC 与设备在同一局域网
4. `hook/device.json` 中的 IP 与设备一致（当前默认 `192.168.130.108`）

---

## 一、环境变量（可选，同一会话内先执行）

> **注意**：下面 3.1 已改为完整 URL，可直接复制运行。若使用 `${BASE}idle` 这类简写，**必须先在本终端执行本段**，否则 PowerShell 会把 `idle` 当成主机名报错「未能解析此远程名称」。

```powershell
$IP       = "192.168.130.108"   # 改成你设备实际 IP
$HOOK     = "D:/Desktop/AI/clawd-micho/clawd-mochi/hook/clawd-hook.js"
$NODE     = "D:\nodejs\node.exe"
$BASE     = "http://$IP/status?s="
```

连通性探测：

```powershell
Invoke-WebRequest -Uri "http://192.168.130.108/state" -UseBasicParsing -TimeoutSec 3 | Select-Object StatusCode, Content
```

期望：返回 JSON，含 `mstate` 字段。

## 二、状态一览

| 状态 | 屏幕表现 | 持续时间说明 |
|------|----------|--------------|
| `idle` | 情绪引擎按心情轮播表情子集（专注/疲惫/雀跃/惬意） | 持续，按心情节奏换表情 |
| `thinking` | 微眯眼 + 上方省略号 `···` 依次起跳 | 持续，直到收到新状态或超时 |
| `working` | 双眼快速左右扫视 | 持续，直到收到新状态或超时 |
| `done` | 闪光庆祝动画 → 自动回到 `idle` | 约 1–2 秒后回到 idle |
| `alert` | 警觉眼 + 上方 "!" 角标，无人处理会逐级加急（温和 0–8s → 催促 8–20s → 告急 20s+ 加描边脉冲） | 持续加急，直到收到新状态（不自动超时） |
| `offline` | 普通眼 + **背光关闭** | 持续，直到收到 `idle` 等开背光状态 |

> **超时保护**：`thinking` / `working` 若 30 秒内无新事件，设备会自动回到 `idle`。`alert` 例外——不自动超时，而是逐级加急直到收到新状态。

## 三、方式 A — 直接 HTTP（推荐，最直观）

每条命令间隔 **3–5 秒**，便于观察屏幕变化。

### 3.1 逐个状态（完整 URL，可直接运行）

将 `192.168.130.108` 换成你的设备 IP。

```powershell
# 1. idle — 空闲
Invoke-WebRequest -Uri "http://192.168.130.108/status?s=idle" -UseBasicParsing

# 2. thinking — 思考/微眯眼 + 省略号
Invoke-WebRequest -Uri "http://192.168.130.108/status?s=thinking" -UseBasicParsing

# 3. working — 工作中/扫视
Invoke-WebRequest -Uri "http://192.168.130.108/status?s=working" -UseBasicParsing

# 4. done — 完成庆祝（会自动回到 idle）
Invoke-WebRequest -Uri "http://192.168.130.108/status?s=done" -UseBasicParsing

# 5. alert — 告警/警觉眼 + "!" 角标
Invoke-WebRequest -Uri "http://192.168.130.108/status?s=alert" -UseBasicParsing

# 6. offline — 离线/关背光
Invoke-WebRequest -Uri "http://192.168.130.108/status?s=offline" -UseBasicParsing

# 7. 恢复 idle（offline 后需手动唤醒）
Invoke-WebRequest -Uri "http://192.168.130.108/status?s=idle" -UseBasicParsing
```

### 3.2 一键顺序跑完全流程

```powershell
$states = @("idle", "thinking", "working", "done", "alert", "offline", "idle")
foreach ($s in $states) {
  Write-Host ">>> $s" -ForegroundColor Cyan
  try {
    $r = Invoke-WebRequest -Uri "${BASE}$s" -UseBasicParsing -TimeoutSec 3
    Write-Host "    OK $($r.Content)"
  } catch {
    Write-Host "    FAIL: $_" -ForegroundColor Red
  }
  Start-Sleep -Seconds 4
}
Write-Host "全状态顺序测试完成" -ForegroundColor Green
```

### 3.3 curl 等价命令（Git Bash / WSL）

```bash
IP=192.168.130.108
for s in idle thinking working done alert offline idle; do
  echo ">>> $s"
  curl -s "http://$IP/status?s=$s"
  echo
  sleep 4
done
```

期望每次响应：`{"ok":1}`

## 四、方式 B — Hook 脚本（Claude Code 事件名）

模拟 Claude Code 通过 argv 传事件：

```powershell
# idle
& $NODE $HOOK SessionStart

# thinking
& $NODE $HOOK UserPromptSubmit

# working
& $NODE $HOOK PreToolUse

# done
& $NODE $HOOK Stop

# alert
& $NODE $HOOK Notification

# offline
& $NODE $HOOK SessionEnd

# 回到 idle
& $NODE $HOOK PostCompact
```

### Claude Code 全事件映射表

| 测试命令 | 映射状态 | 屏幕 |
|----------|----------|------|
| `SessionStart` | idle | 空闲 |
| `PostCompact` | idle | 空闲 |
| `UserPromptSubmit` | thinking | 微眯眼 + 省略号 `···` |
| `PreCompact` | thinking | 微眯眼 + 省略号 `···` |
| `PreToolUse` | working | 扫视 |
| `PostToolUse` | working | 扫视 |
| `SubagentStart` | working | 扫视 |
| `SubagentStop` | working | 扫视 |
| `Stop` | done | 庆祝 → idle |
| `Notification` | alert | 警觉眼 + "!" 角标（经 `notificationIsAlert` 过滤待命提醒） |
| `Elicitation` | alert | 警觉眼 + "!" 角标 |
| `PostToolUseFailure` | working | 保持工作态（不再弹 alert） |
| `StopFailure` | idle | 回 idle |
| `SessionEnd` | offline | 关背光 |

### Claude Code 顺序测试脚本

```powershell
$events = @(
  "SessionStart",
  "UserPromptSubmit",
  "PreToolUse",
  "Stop",
  "Notification",
  "SessionEnd",
  "PostCompact"
)
foreach ($e in $events) {
  Write-Host ">>> Claude: $e" -ForegroundColor Cyan
  & $NODE $HOOK $e
  Start-Sleep -Seconds 4
}
```

## 五、方式 C — Hook 脚本（Cursor 事件名）

Cursor 通过 **stdin JSON** 传 `hook_event_name`：

```powershell
# idle
'{"hook_event_name":"sessionStart"}' | & $NODE $HOOK

# thinking（会输出 {"continue":true} 到 stdout，属正常）
'{"hook_event_name":"beforeSubmitPrompt"}' | & $NODE $HOOK

# working
'{"hook_event_name":"preToolUse"}' | & $NODE $HOOK

# done
'{"hook_event_name":"stop"}' | & $NODE $HOOK

# working（工具失败,不再是 alert；Cursor 侧已无事件触发 alert,测 alert 请用直接 HTTP /status?s=alert）
'{"hook_event_name":"postToolUseFailure"}' | & $NODE $HOOK

# offline
'{"hook_event_name":"sessionEnd"}' | & $NODE $HOOK

# 回到 idle
'{"hook_event_name":"sessionStart"}' | & $NODE $HOOK
```

### Cursor 全事件映射表

| stdin `hook_event_name` | 映射状态 | 屏幕 |
|-------------------------|----------|------|
| `sessionStart` | idle | 空闲 |
| `beforeSubmitPrompt` | thinking | 微眯眼 + 省略号 `···` |
| `preCompact` | thinking | 微眯眼 + 省略号 `···` |
| `preToolUse` | working | 扫视 |
| `postToolUse` | working | 扫视 |
| `subagentStart` | working | 扫视 |
| `subagentStop` | working | 扫视 |
| `stop` | done | 庆祝 → idle |
| `postToolUseFailure` | working | 保持工作态（不再弹 alert） |
| `sessionEnd` | offline | 关背光 |

> 未映射事件（如 `afterAgentThought`）会静默跳过，设备无变化。

### Cursor 顺序测试脚本

```powershell
$cursorEvents = @(
  "sessionStart",
  "beforeSubmitPrompt",
  "preToolUse",
  "stop",
  "postToolUseFailure",
  "sessionEnd",
  "sessionStart"
)
foreach ($e in $cursorEvents) {
  Write-Host ">>> Cursor: $e" -ForegroundColor Cyan
  "{`"hook_event_name`":`"$e`"}" | & $NODE $HOOK
  Start-Sleep -Seconds 4
}
```

## 六、典型会话流程模拟

模拟一次完整的 Agent 对话生命周期：

```powershell
Write-Host "=== 模拟 Agent 会话 ===" -ForegroundColor Yellow

# 1. 打开 Cursor / Claude → sessionStart
& $NODE $HOOK SessionStart
Start-Sleep -Seconds 3

# 2. 用户发送消息 → thinking
& $NODE $HOOK UserPromptSubmit
Start-Sleep -Seconds 3

# 3. Agent 调用工具 → working
& $NODE $HOOK PreToolUse
Start-Sleep -Seconds 3

# 4. 工具返回 → 仍在 working
& $NODE $HOOK PostToolUse
Start-Sleep -Seconds 3

# 5. Agent 回复完成 → done → 自动 idle
& $NODE $HOOK Stop
Start-Sleep -Seconds 5

# 6. 关闭 IDE → offline
& $NODE $HOOK SessionEnd
Start-Sleep -Seconds 3

# 7. 重新打开 → idle
& $NODE $HOOK SessionStart
Write-Host "=== 会话模拟完成 ===" -ForegroundColor Green
```

Cursor 版（把事件名换成 stdin 形式）：

```powershell
function Invoke-CursorHook($name) {
  "{`"hook_event_name`":`"$name`"}" | & $NODE $HOOK
}

Invoke-CursorHook "sessionStart";       Start-Sleep 3
Invoke-CursorHook "beforeSubmitPrompt"; Start-Sleep 3
Invoke-CursorHook "preToolUse";         Start-Sleep 3
Invoke-CursorHook "postToolUse";        Start-Sleep 3
Invoke-CursorHook "stop";               Start-Sleep 5
Invoke-CursorHook "sessionEnd";         Start-Sleep 3
Invoke-CursorHook "sessionStart"
```

- [ ] `idle` — 屏幕有表情，会按当前心情轮换该心情子集内的表情
- [ ] `thinking` — 微眯眼 + 上方省略号 `···` 依次起跳
- [ ] `working` — 双眼左右扫视动画
- [ ] `done` — 闪光庆祝后自动回到 idle
- [ ] `alert` — 警觉眼 + 上方 "!" 角标
- [ ] `offline` — 背光熄灭
- [ ] `idle`（再次）— 背光重新亮起
- [ ] HTTP 每次返回 `{"ok":1}`
- [ ] Claude Code Hook 7 事件顺序脚本跑通
- [ ] Cursor Hook 7 事件顺序脚本跑通
- [ ] 30 秒无事件后 thinking/working 自动回 idle

## 九、故障排查

| 现象 | 检查项 |
|------|--------|
| 全部 FAIL / 超时 | IP 是否正确；设备是否连 WiFi；PC 是否同网段 |
| HTTP 通但 Hook 无效 | `device.json` IP；`node.exe` 路径；Cursor/Claude hooks.json 路径 |
| 只有部分状态有动画 | 是否已烧录最新固件；是否卡在 Draw/Code 视图（Hook 不会打断） |
| `done` 后无庆祝 | 确认发送的是 `done` 而非 `idle` |
| `offline` 后无法唤醒 | 需再发 `idle` / `SessionStart` / `sessionStart` |
| Cursor Hook 无反应 | Settings → Hooks 是否加载；重启 Cursor；检查 hooks.json 是否被 Clawd on Desk 覆盖 |

临时指定 IP（不改 device.json）：

```powershell
$env:CLAWD_DEVICE_IP = "192.168.130.108"
& $NODE $HOOK SessionStart
```

## 十二、working 子状态与 Ticker 测试（需设备）

```powershell
$T = "clawd.local"   # 也可换成设备实际 IP，如 "192.168.130.108"

# 1. read — 低头扫读；ticker 显示 "> read <文件名>"
Invoke-WebRequest -Uri "http://$T/status?s=working&act=read&info=main.cpp" -UseBasicParsing
# 预期：眼睛下移缓慢左右扫读，底部绿色 ticker 显示 "> read main.cpp"

Start-Sleep 3

# 2. edit — 打字机；眼随写入点追行，底部代码行逐格打出，含暗色痕迹
Invoke-WebRequest -Uri "http://$T/status?s=working&act=edit&info=App.tsx" -UseBasicParsing
# 预期：眼左偏跟随写入点从左向右移动，底部出现绿色代码块逐格增长

Start-Sleep 3

# 3. run — 紧绷快速小扫视；ticker 显示 "> run <命令首词>"
Invoke-WebRequest -Uri "http://$T/status?s=working&act=run&info=npm" -UseBasicParsing
# 预期：眼快速随机小幅抖动，ticker 显示 "> run npm"

Start-Sleep 3

# 4. net — 大幅张望；眼在左右大幅来回
Invoke-WebRequest -Uri "http://$T/status?s=working&act=net&info=github.com" -UseBasicParsing
# 预期：眼在左右 ±24 像素大幅跳望，ticker 显示 "> net github.com"

Start-Sleep 3

# 5. agent — 经典扫视（与 work 相同）；ticker 显示 "> agent <描述>"
Invoke-WebRequest -Uri "http://$T/status?s=working&act=agent&info=explore" -UseBasicParsing
# 预期：经典左右扫视，ticker 显示 "> agent explore"

Start-Sleep 3

# 6. work — 通用经典扫视
Invoke-WebRequest -Uri "http://$T/status?s=working&act=work&info=ToolSearch" -UseBasicParsing
# 预期：经典左右扫视，ticker 显示 "> work ToolSearch"

Start-Sleep 3

# 7. 超长 info — 跑马灯（超过 19 列时 150ms 滚动）
Invoke-WebRequest -Uri "http://$T/status?s=working&act=read&info=very-long-file-name-that-exceeds-limit.cpp" -UseBasicParsing
# 预期：底部 ticker 超出宽度时自动向左滚动，每 150ms 移一格

Start-Sleep 4

# 8. 未知 act — 兜底为经典扫视（work），不报错
Invoke-WebRequest -Uri "http://$T/status?s=working&act=unknown&info=test" -UseBasicParsing
# 预期：经典扫视，ticker 显示 "> work test"

Start-Sleep 3

# 9. 裸 working（无 act/info）— 兼容老 hook；经典扫视，无 ticker
Invoke-WebRequest -Uri "http://$T/status?s=working" -UseBasicParsing
# 预期：经典扫视，底部 ticker 区域消失（不显示任何文字）

Start-Sleep 3

# 恢复 idle
Invoke-WebRequest -Uri "http://$T/status?s=idle" -UseBasicParsing
```
