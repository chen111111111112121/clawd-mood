## 1. 项目简介

Clawd Mochi 是一款桌面物理伴侣：ESP32-C3 驱动 240×240 彩屏，显示 Clawd 像素表情。

**两种使用方式：**

| 模式 | 说明 |
|------|------|
| **手动控制** | 手机连设备热点，浏览器打开控制器，切换表情 / 绘图 |
| **Monitor 监测** | PC 上 Claude Code 通过 Hook 推送状态，屏幕自动切换表情 |

Monitor 模式下 PC **不需要**连接设备热点，只要和设备在同一局域网即可，PC 可正常上网。

## 2. 硬件准备与接线

### 2.1 物料清单

| 部件 | 规格 | 参考价 |
|------|------|--------|
| ESP32-C3 Super Mini | 带 WiFi | ~¥18 |
| ST7789 1.54" TFT | 240×240 SPI | ~¥22 |
| 杜邦线 × 8 | 8–10 cm | ~¥4 |
| USB-C 线 | 供电 | — |

### 2.2 接线（以固件为准）

> ⚠️ VCC 必须接 **3.3V**，禁止接 5V。

| 屏幕引脚 | ESP32-C3 GPIO | 说明 |
|----------|---------------|------|
| VCC | 3V3 | 电源 |
| GND | GND | 地 |
| SDA | GPIO 9 | SPI MOSI |
| SCL | GPIO 8 | SPI SCK |
| RES | GPIO 2 | 复位 |
| DC | GPIO 1 | 数据/命令 |
| CS | GPIO 4 | 片选 |
| BL | GPIO 3 | 背光 |

### 2.3 上电验证

USB-C 供电后，屏幕应依次显示：

1. 开机招呼动画：眼睛缓缓睁开，随即俏皮 wink + 上扬微笑 + 一颗白色小星（纯图形，无文字/无 logo，风格与设备表情一致）
2. 开机信息屏（`VIEW_BOOTINFO`）：
   - **连上家里 WiFi**：显示局域网 IP，约 3 秒后自动切入表情界面
   - **未连上**（没配过 / 旧凭据失效 / 换了网络环境）：常驻信息屏，并保持 AP 热点稳定，供你随时连入重新配网；连不上时设备**不会**死磕旧网络（会停掉 STA 重连以免搅乱热点）

## 3. 烧录固件

### 3.1 安装 Arduino IDE

下载 [Arduino IDE 2.x](https://www.arduino.cc/en/software) 并安装。

### 3.2 添加 ESP32 支持

1. **文件 → 首选项**，在「附加开发板管理器网址」填入：

   ```
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
   ```

2. **工具 → 开发板 → 开发板管理器**，搜索 `esp32`，安装 **esp32 by Espressif Systems**。

### 3.3 安装依赖库

**工具 → 管理库**，安装：

- `Adafruit GFX Library`
- `Adafruit ST7735 and ST7789 Library`

### 3.4 开发板设置

| 设置项 | 值 |
|--------|-----|
| 开发板 | ESP32C3 Dev Module |
| USB CDC On Boot | **Enabled**（重要） |
| CPU Frequency | 160 MHz |
| Upload Speed | 921600 |

### 3.5 上传

1. 打开 `clawd_mood/clawd_mood.ino`
2. USB-C 连接 ESP32，选择正确端口
3. 点击 **上传**，等待 `Hard resetting via RTS pin...` 表示成功

## 4. 设备 WiFi 配置

设备支持 **双 WiFi 模式**：

- **AP 热点**（始终开启）：手机随时可连入控制
- **STA 客户端**：连家里 WiFi，供 PC Hook 访问

### 4.1 配置步骤

1. 手机 WiFi 设置，连接 **`ClaWD-Mood`**，密码 **`clawd1234`**
2. 浏览器打开 **`http://192.168.4.1`**
3. 页面下方找到 **wifi setup** 区域
4. 填入家里 WiFi 的 **SSID** 和 **密码**
5. 点击 **Save & Connect**
6. 连接成功后：
   - 页面显示 `Connected: 192.168.x.x`，并自动给出 PC 安装命令（`.\install-global.ps1 -DeviceIP <ip>`）
   - 设备屏幕显示 **Home WiFi** 及分配的 IP，约 3 秒后自动回到表情界面（不会卡在信息屏）

### 4.2 记下设备 IP

示例：`192.168.130.108`

此 IP 用于后续 Hook 配置。凭证保存在设备 Flash 中，重启后自动重连。

> **不记得 IP？** 安装脚本不带参数可自动发现 `clawd.local`；也可点 Web 控制器的 **`ℹ device info`** 按钮（或访问 `/cmd?k=i`）让设备屏幕再次显示 IP 信息。

### 4.3 网络拓扑

```
┌─────────────┐                    ┌─────────────┐
│   手机      │── 连热点 ─────────→│  ESP32      │
│  (可选)     │   192.168.4.1      │  Clawd Mochi│
└─────────────┘                    └──────┬──────┘
                                        │ 家里 WiFi
┌─────────────┐   HTTP /status        │
│   PC        │──────────────────────→│ 192.168.x.x
│ Claude Code │                       │
└─────────────┘                    └─────────────┘
```

## 5. Web 控制器使用

连设备热点或同一局域网均可访问控制器（AP 模式下固定为 `http://192.168.4.1`）。

### 5.1 视图模式

| 按钮 / 视图 | 固件视图常量 | 功能 |
|-------------|-------------|------|
| Normal eyes | `VIEW_EYES_NORMAL` | 方形眼，左右摆动 + 眨眼 |
| Squish eyes | `VIEW_EYES_SQUISH` | `> <` 眯眼开合动画 |
| Claude Code | `VIEW_CODE` | 显示标题 + 可输入伪终端 |
| Canvas | `VIEW_DRAW` | 手机触摸绘图，同步到屏幕 |
| **Monitor** | `VIEW_MONITOR` | 进入监测模式，接收 Hook 状态 |
| *(开机 / 召回)* | `VIEW_BOOTINFO` | 开机信息屏（WiFi / IP）；连上 3 秒后进表情，未连上则常驻并保持 AP 稳定供配网；可通过 `/cmd?k=i` 或 `ℹ device info` 按钮召回 |

### 5.2 其他控制

| 控制项 | 说明 |
|--------|------|
| Display on/off | 开关背光 |
| ℹ device info | 召回开机信息屏（显示 WiFi / IP），调用 `/cmd?k=i` |
| Speed | 动画速度：慢 / 正常 / 快 |
| Background | 背景颜色 |
| Pen color | Canvas 画笔颜色 |

### 5.3 Monitor 面板

进入 Monitor 模式后显示：

- **MONITOR MODE** — 当前监测状态
- **DEVICE IP** — 设备局域网 IP
- **PC LINK** — 是否收到 Hook 心跳
- **LAST EVENT** — 最后一次状态事件

## 6. Claude Code Hook 配置

### 6.1 前置条件

- [ ] 固件已烧录（含 `/status` 端点）
- [ ] 设备已连上家里 WiFi（开机信息屏显示 IP 后约 3 秒进表情，即为成功）
- [ ] PC 已安装 [Node.js](https://nodejs.org/)
- [ ] PC 与设备在同一局域网

### 6.2 一键安装（推荐）

在 `hook/` 目录执行：

```powershell
cd hook
.\install-global.ps1
```

脚本会自动尝试发现 `clawd.local`；也可手动指定 IP 或主机名：

```powershell
.\install-global.ps1 -DeviceIP 192.168.130.108
.\install-global.ps1 -DeviceIP clawd.local
```

> **WiFi 配置完成后门户页也会直接给出安装命令**，复制粘贴到 PC 终端即可。

安装后 IP 解析优先级：环境变量 `CLAWD_DEVICE_IP` > `%USERPROFILE%\.clawd-mood\hook\device.json` > 脚本内置 `clawd.local`（mDNS）。IP 变更时只需重跑安装脚本或直接编辑 `device.json`。

### 6.3 手动配置设备 IP

若需手动编辑，修改已安装的 `%USERPROFILE%\.clawd-mood\hook\device.json`：

```json
{
  "device_ip": "192.168.130.108"
}
```

也可用环境变量（优先级更高）：

```powershell
$env:CLAWD_DEVICE_IP = "192.168.130.108"
```

### 6.4 测试 Hook

PowerShell 中执行（按你的 Node.js 安装路径调整）：

```powershell
& "D:\nodejs\node.exe" "D:/Desktop/AI/clawd-micho/clawd-mochi/hook/clawd-hook.js" UserPromptSubmit
```

设备应切换为 **思考（thinking）**：微眯眼 + 上方省略号 `···`。

```powershell
& "D:\nodejs\node.exe" "D:/Desktop/AI/clawd-micho/clawd-mochi/hook/clawd-hook.js" Stop
```

设备应回到 **普通眼（idle）**。

## 7. 监测状态说明

> 多工具：PC 控制台（`agent/`）绑定"当前响应哪款 AI 工具"，hook 读 `~/.clawd-mood/agent.json` 的 `activeTool` 自我门控，非当前工具静默忽略。控制台 `node agent/clawd-agent.js` → `http://127.0.0.1:6624`。

Hook 通过 `/status?s=...` 推送，共 **6 种状态**：

| 状态 | 参数 | 屏幕 | 触发事件 |
|------|------|------|----------|
| 空闲 | `idle` | 情绪引擎按心情轮播表情子集（专注/疲惫/雀跃/惬意） | SessionStart、Stop |
| 思考 | `thinking` | 微眯眼 + 省略号 `···` | UserPromptSubmit |
| 工作 | `working` | 按工具显示六种姿态（读/写/跑/搜/子代理/通用），底部 ticker 显示当前文件或命令 | PreToolUse、PostToolUse |
| 提醒 | `alert` | 警觉眼 + "!" 角标，无人处理逐级加急（温和 0–8s → 催促 8–20s → 告急 20s+ 描边） | Notification / Elicitation（普通工具失败不再触发） |
| 离线 | `offline` | 普通眼 + 关背光 | SessionEnd |

**超时保护：** `thinking` / `working` 状态下，若 30 秒内无新事件，自动回到 `idle`。

## 8. API 参考

设备 IP 以 `192.168.130.108` 为例。

### Hook 推送状态

```
GET http://192.168.130.108/status?s=idle
GET http://192.168.130.108/status?s=thinking
GET http://192.168.130.108/status?s=working
GET http://192.168.130.108/status?s=working&act=read|edit|run|net|agent|work&info=<短文本>
GET http://192.168.130.108/status?s=alert
GET http://192.168.130.108/status?s=offline
```

`working` 时可附带工具语义参数（`act`/`info`），显示对应姿态与底部 ticker；省略则为经典扫视，兼容老 hook。

设备同时注册 `http://clawd.local`（mDNS），IP 变化后无需修改配置。

响应：`{"ok":1}`

### 查询设备状态

```
GET http://192.168.130.108/state
```

响应示例：

```json
{
  "view": 4,
  "busy": false,
  "term": false,
  "bl": true,
  "speed": 1,
  "mstate": "thinking",
  "sta": true,
  "sta_ip": "192.168.130.108",
  "monitor": true,
  "last_status": 3
}
```

### WiFi 配置（Web 控制器调用）

```
GET http://192.168.4.1/wifi/save?ssid=你的WiFi名&pass=你的密码
```

### 手动控制命令（`/cmd`）

| 命令 | 说明 |
|------|------|
| `/cmd?k=i` | 召回开机信息屏（`VIEW_BOOTINFO`），显示 WiFi / IP，常驻直到下次切视图 |
| `/cmd?k=m` | 切换到 Monitor 模式 |
| `/cmd?k=a` | 重放开机招呼动画（睁眼 + wink 打招呼）后回到普通眼表情 |

```
GET http://192.168.130.108/cmd?k=i
```

|------|------|
| `http://192.168.4.1` | 连设备热点时 |

## 9. 故障排查

### 设备 / 网络

| 现象 | 排查 |
|------|------|
| 屏幕不亮 | 检查 3.3V 接线、USB 供电 |
| 手机搜不到热点 | 若设备配过的网络已失效（换了环境），开机后等约 10 秒让 STA 尝试结束，AP 即稳定；设备会自动停在信息屏供你连入重配 |
| 家里 WiFi 连不上 | SSID/密码是否正确；是否为 2.4GHz（ESP32 不支持 5GHz 专用网络）。连不上时设备停在信息屏常驻，连 AP 重新配网即可 |
| IP 变了 | 重新查看屏幕或 Web 页面，更新 `device.json` |

### Hook / Claude Code

| 现象 | 排查 |
|------|------|
| Hook 无反应 | 浏览器能否打开 `http://<设备IP>/`；IP 是否正确 |
| 状态不更新 | 是否在 Canvas/终端模式（会屏蔽 Hook）；可手动点 Monitor |
| 30 秒后变 idle | 正常现象，超时保护 |
| Claude Code 报错 | Node.js 路径是否正确；`settings.json` JSON 格式是否有效 |
| PC 能上网但 Hook 不通 | 防火墙是否拦截；PC 与设备是否同网段 |

### 手动测试命令

```powershell
# 测试设备是否在线
curl http://192.168.130.108/state

# 手动推送状态
curl "http://192.168.130.108/status?s=thinking"
```

Hook 脚本设计为**静默失败**，不会影响 Claude Code 正常使用。
