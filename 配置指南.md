## 1. 项目简介

Clawd Mochi 是一款桌面物理伴侣：ESP32-C3 驱动 240×240 彩屏，显示 Clawd 像素表情。

**两种使用方式：**

| 模式 | 说明 |
|------|------|
| **手动控制** | 手机连设备热点，浏览器打开控制器，切换表情 / 绘图 |
| **Monitor 监测** | PC 上 Claude Code 通过 Hook 推送状态，屏幕自动切换表情 |

Monitor 模式下 PC **不需要**连接设备热点，只要和设备在同一局域网即可，PC 可正常上网。

## 2. 硬件准备与接线

### 2.1 物料清单

| 部件 | 规格 | 参考价 |
|------|------|--------|
| ESP32-C3 Super Mini | 带 WiFi | ~¥18 |
| ST7789 1.54" TFT | 240×240 SPI | ~¥22 |
| 杜邦线 × 8 | 8–10 cm | ~¥4 |
| USB-C 线 | 供电 | — |

### 2.2 接线（以固件为准）

> ⚠️ VCC 必须接 **3.3V**，禁止接 5V。

| 屏幕引脚 | ESP32-C3 GPIO | 说明 |
|----------|---------------|------|
| VCC | 3V3 | 电源 |
| GND | GND | 地 |
| SDA | GPIO 9 | SPI MOSI |
| SCL | GPIO 8 | SPI SCK |
| RES | GPIO 2 | 复位 |
| DC | GPIO 1 | 数据/命令 |
| CS | GPIO 4 | 片选 |
| BL | GPIO 3 | 背光 |

### 2.3 上电验证

USB-C 供电后，屏幕应依次显示：

1. `Clawd Mochi` 开屏文字
2. Logo 逐线动画
3. WiFi 连接信息（热点名 + 家里 WiFi 状态 + IP）

## 3. 烧录固件

### 3.1 安装 Arduino IDE

下载 [Arduino IDE 2.x](https://www.arduino.cc/en/software) 并安装。

### 3.2 添加 ESP32 支持

1. **文件 → 首选项**，在「附加开发板管理器网址」填入：

   ```
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
   ```

2. **工具 → 开发板 → 开发板管理器**，搜索 `esp32`，安装 **esp32 by Espressif Systems**。

### 3.3 安装依赖库

**工具 → 管理库**，安装：

- `Adafruit GFX Library`
- `Adafruit ST7735 and ST7789 Library`

### 3.4 开发板设置

| 设置项 | 值 |
|--------|-----|
| 开发板 | ESP32C3 Dev Module |
| USB CDC On Boot | **Enabled**（重要） |
| CPU Frequency | 160 MHz |
| Upload Speed | 921600 |

### 3.5 上传

1. 打开 `clawd_mochi/clawd_mochi.ino`
2. USB-C 连接 ESP32，选择正确端口
3. 点击 **上传**，等待 `Hard resetting via RTS pin...` 表示成功

## 4. 设备 WiFi 配置

设备支持 **双 WiFi 模式**：

- **AP 热点**（始终开启）：手机随时可连入控制
- **STA 客户端**：连家里 WiFi，供 PC Hook 访问

### 4.1 配置步骤

1. 手机 WiFi 设置，连接 **`ClaWD-Mochi`**，密码 **`clawd1234`**
2. 浏览器打开 **`http://192.168.4.1`**
3. 页面下方找到 **wifi setup** 区域
4. 填入家里 WiFi 的 **SSID** 和 **密码**
5. 点击 **Save & Connect**
6. 连接成功后：
   - 页面显示 `Connected: 192.168.x.x`
   - 设备屏幕显示 **Home WiFi** 及分配的 IP

### 4.2 记下设备 IP

示例：`192.168.130.108`

此 IP 用于后续 Hook 配置。凭证保存在设备 Flash 中，重启后自动重连。

### 4.3 网络拓扑

```
┌─────────────┐                    ┌─────────────┐
│   手机      │── 连热点 ─────────→│  ESP32      │
│  (可选)     │   192.168.4.1      │  Clawd Mochi│
└─────────────┘                    └──────┬──────┘
                                        │ 家里 WiFi
┌─────────────┐   HTTP /status        │
│   PC        │──────────────────────→│ 192.168.x.x
│ Claude Code │                       │
└─────────────┘                    └─────────────┘
```

## 5. Web 控制器使用

连设备热点或同一局域网均可访问控制器（AP 模式下固定为 `http://192.168.4.1`）。

### 5.1 视图模式

| 按钮 | 功能 |
|------|------|
| Normal eyes | 方形眼，左右摆动 + 眨眼 |
| Squish eyes | `> <` 眯眼开合动画 |
| Claude Code | 显示标题 + 可输入伪终端 |
| Canvas | 手机触摸绘图，同步到屏幕 |
| **Monitor** | 进入监测模式，接收 Hook 状态 |

### 5.2 其他控制

| 控制项 | 说明 |
|--------|------|
| Display on/off | 开关背光 |
| Speed | 动画速度：慢 / 正常 / 快 |
| Background | 背景颜色 |
| Pen color | Canvas 画笔颜色 |

### 5.3 Monitor 面板

进入 Monitor 模式后显示：

- **MONITOR MODE** — 当前监测状态
- **DEVICE IP** — 设备局域网 IP
- **PC LINK** — 是否收到 Hook 心跳
- **LAST EVENT** — 最后一次状态事件

## 6. Claude Code Hook 配置

### 6.1 前置条件

- [ ] 固件已烧录（含 `/status` 端点）
- [ ] 设备已连上家里 WiFi，IP 已知
- [ ] PC 已安装 [Node.js](https://nodejs.org/)
- [ ] PC 与设备在同一局域网

### 6.2 配置设备 IP

编辑 `hook/device.json`：

```json
{
  "device_ip": "192.168.130.108"
}
```

将 IP 替换为你设备屏幕上显示的实际地址。

也可用环境变量（优先级更高）：

```powershell
$env:CLAWD_DEVICE_IP = "192.168.130.108"
```

### 6.3 测试 Hook

PowerShell 中执行（按你的 Node.js 安装路径调整）：

```powershell
& "D:\nodejs\node.exe" "D:/Desktop/AI/clawd-micho/clawd-mochi/hook/clawd-hook.js" UserPromptSubmit
```

设备应切换为 **眯眼（thinking）**。

```powershell
& "D:\nodejs\node.exe" "D:/Desktop/AI/clawd-micho/clawd-mochi/hook/clawd-hook.js" Stop
```

设备应回到 **普通眼（idle）**。

### 6.6 端到端验证清单

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 启动 Claude Code 新会话 | 设备 → idle（普通眼） |
| 2 | 发送一条消息 | 设备 → thinking（眯眼） |
| 3 | Agent 调用工具 | 设备 → working（按工具六种姿态 + 底部 ticker） |
| 4 | 任务完成 | 设备 → idle |
| 5 | 收到通知 | 设备 → alert（Logo 动画） |
| 6 | 关闭会话 | 设备 → offline（关背光） |

## 7. 监测状态说明

Hook 通过 `/status?s=...` 推送，共 **6 种状态**：

| 状态 | 参数 | 屏幕 | 触发事件 |
|------|------|------|----------|
| 空闲 | `idle` | 普通眼 | SessionStart、Stop |
| 思考 | `thinking` | 眯眼 `> <` | UserPromptSubmit |
| 工作 | `working` | 按工具显示六种姿态（读/写/跑/搜/子代理/通用），底部 ticker 显示当前文件或命令 | PreToolUse、PostToolUse |
| 提醒 | `alert` | Logo 动画 | Notification |
| 离线 | `offline` | 普通眼 + 关背光 | SessionEnd |

**超时保护：** `thinking` / `working` 状态下，若 30 秒内无新事件，自动回到 `idle`。

## 8. API 参考

设备 IP 以 `192.168.130.108` 为例。

### Hook 推送状态

```
GET http://192.168.130.108/status?s=idle
GET http://192.168.130.108/status?s=thinking
GET http://192.168.130.108/status?s=working
GET http://192.168.130.108/status?s=working&act=read|edit|run|net|agent|work&info=<短文本>
GET http://192.168.130.108/status?s=alert
GET http://192.168.130.108/status?s=offline
```

`working` 时可附带工具语义参数（`act`/`info`），显示对应姿态与底部 ticker；省略则为经典扫视，兼容老 hook。

设备同时注册 `http://clawd.local`（mDNS），IP 变化后无需修改配置。

响应：`{"ok":1}`

### 查询设备状态

```
GET http://192.168.130.108/state
```

响应示例：

```json
{
  "view": 4,
  "busy": false,
  "term": false,
  "bl": true,
  "speed": 1,
  "mstate": "thinking",
  "sta": true,
  "sta_ip": "192.168.130.108",
  "monitor": true,
  "last_status": 3
}
```

### WiFi 配置（Web 控制器调用）

```
GET http://192.168.4.1/wifi/save?ssid=你的WiFi名&pass=你的密码
```

### Web 控制器

| 地址 | 场景 |
|------|------|
| `http://192.168.4.1` | 连设备热点时 |
| `http://192.168.130.108` | 同一局域网内 |

## 9. 故障排查

### 设备 / 网络

| 现象 | 排查 |
|------|------|
| 屏幕不亮 | 检查 3.3V 接线、USB 供电 |
| 连不上热点 | 重启设备，等 3 秒开机动画结束 |
| 家里 WiFi 连不上 | SSID/密码是否正确；是否为 2.4GHz（ESP32 不支持 5GHz 专用网络） |
| IP 变了 | 重新查看屏幕或 Web 页面，更新 `device.json` |

### Hook / Claude Code

| 现象 | 排查 |
|------|------|
| Hook 无反应 | 浏览器能否打开 `http://<设备IP>/`；IP 是否正确 |
| 状态不更新 | 是否在 Canvas/终端模式（会屏蔽 Hook）；可手动点 Monitor |
| 30 秒后变 idle | 正常现象，超时保护 |
| Claude Code 报错 | Node.js 路径是否正确；`settings.json` JSON 格式是否有效 |
| PC 能上网但 Hook 不通 | 防火墙是否拦截；PC 与设备是否同网段 |

### 手动测试命令

```powershell
# 测试设备是否在线
curl http://192.168.130.108/state

# 手动推送状态
curl "http://192.168.130.108/status?s=thinking"
```

Hook 脚本设计为**静默失败**，不会影响 Claude Code 正常使用。
